# 充电器升级上位机

这是一个用于替代 SecureCRT 手工升级流程的 Windows 上位机工具。用户只需要连接串口、点击下载并选择固件文件，然后给充电器上 AC 电，上位机会自动发送 `update` 并通过 YMODEM 协议发送固件。

## 交付目标

- 运行端不需要安装 Python、.NET、pyserial 或 SecureCRT。
- 编译输出为 32 位 Windows exe，优先兼容老 Windows。
- 串口参数固定为 `115200 8N1`，避免用户手工配置错误。
- 串口日志按 GB2312/CP936 解码显示，兼容原 SecureCRT 使用方式。
- 前期调试阶段会把 bootloader 提示文字、`update` 触发点、进入“等待文件发送”状态等信息实时打印到日志框。
- 第一次点击“下载”时选择固件，随后进入批量待机模式；同一批次内固件路径会复用。
- 识别到 bootloader 的“输入 `update` 开始升级程序”提示后自动发送 `update`。
- 收到“等待文件发送……”提示后再进入 YMODEM 发送流程。
- 等待充电器进入 YMODEM 接收状态后自动发送 `.bin` 固件。
- 单台成功后会自动回到待机状态，继续等下一台设备上电。
- 单台成功后会先收尾并等待串口静默，再重新武装下一次上电识别，避免上一台的成功日志干扰下一台。
- 界面只保留串口连接和下载相关操作。

## 直接运行

当前仓库提供两个 32 位 Windows 程序：

```text
release\ChargerUpdater_debug.exe
release\ChargerUpdater_user.exe
```

- `ChargerUpdater_debug.exe`：保留完整调试日志，适合联调和现场排查
- `ChargerUpdater_user.exe`：面向普通操作员，界面更简化，只强调连接、待机、升级中、升级成功

最终用户只需要对应的 exe，不需要安装 Python、.NET、pyserial 或 SecureCRT。

## 重新生成 exe

当前仓库同时保留原生 Win32 C++ 源码和构建脚本，便于后续维护。

在开发机上构建：

```bat
build_mingw32.bat
```

脚本会优先使用 `C:\qp\qtools\MinGW32\bin\g++.exe`，如果不存在则使用 `PATH` 中的 `g++`。输出文件：

```text
release\ChargerUpdater_debug.exe
release\ChargerUpdater_user.exe
```

## 使用步骤

1. 将 USB 转串口线连接电脑和充电器升级串口。
2. 打开 `ChargerUpdater.exe`，选择对应 COM 口，点击“连接”。
3. 第一次点击“下载”，选择充电器固件 `.bin` 文件。
4. 工具进入批量待机模式后，给充电器接入 AC 电源。
5. 工具识别到 bootloader 提示“输入 `update(小写)` 开始升级程序”后，会在 7 秒窗口内自动发送 `update`。
6. 工具收到“等待文件发送……”提示后自动执行 YMODEM 固件发送。
7. 当前设备升级完成后，工具会自动回到待机状态，继续等待下一台设备上电。

## 注意事项

- 下载过程中不要关闭工具、拔掉串口线或切断充电器电源。
- 必须先点击“下载”进入批量待机，再给充电器上电，这样工具才能稳定卡在 bootloader 的 7 秒升级窗口内自动触发。
- 如果长时间没有进度，请确认选择的 COM 口正确，并重新点击“断开”后再“连接”。
- 如果充电器固件较大，YMODEM 发送会持续一段时间，期间日志可能不会持续滚动，这是正常现象。
- 如果日志里出现“设备已开始加载内部 FLASH 程序，说明 7 秒升级窗口已错过”，说明下载点击太晚、串口未连好，或者设备没有进入预期 bootloader 流程。
- 如果要切换到另一版固件，先点击“断开”，再重新连接并点击“下载”选择新的 `.bin` 文件。

## 批量升级建议

1. 整个批次保持上位机常开，并保持串口已连接。
2. 同一版本固件连续升级时，只在批次开始时选择一次 `.bin` 文件。
3. 每台设备都按“先进入待机、后上电”的顺序操作，不要反过来。
4. 单台成功后无需再次选文件，工具会继续等待下一台。
5. 如果某台设备提示已错过 7 秒窗口，直接断电重来，比长时间等待更快。
