# 充电器升级流程说明

## 旧流程

1. 安装 SecureCRT。
2. 手动创建串口连接，设置波特率 `115200`，编码格式 `GB2312`。
3. 充电器接入 AC 电源后等待串口提示。
4. 手动输入 `update`。
5. 在 SecureCRT 中拖入固件 `.bin` 文件并选择 YMODEM 协议发送。

## 新流程

1. 打开 `release\ChargerUpdater.exe`。
2. 选择 COM 口并点击“连接”。
3. 第一次点击“下载”并选择固件 `.bin` 文件。
4. 工具进入批量待机模式。
5. 给充电器接入 AC 电源。
6. 工具识别到 bootloader 输出中的“输入 `update(小写)` 开始升级程序”提示后，在 7 秒窗口内自动发送 `update`。
7. 设备回送“等待文件发送……”提示后，工具自动等待 YMODEM 握手并发送固件。
8. 当前设备升级完成后，工具自动回到待机模式，继续等待下一台。
9. 工具会先收尾当前设备升级后的串口输出，并等待串口静默后再重新武装下一次识别。

## 协议处理

- 串口参数固定为 `115200 8N1`。
- 串口显示按 `GB2312` 解码。
- 串口和界面均使用 Win32 API 实现，运行端不依赖 Python、.NET、SecureCRT 或 pyserial。
- YMODEM 文件头使用 128 字节包，数据包优先使用 1024 字节包，小于等于 128 字节时使用 128 字节包。
- 数据包使用 CRC16-CCITT 校验。
- 发送完成后按 YMODEM batch 传输流程发送 EOT 和空文件头。

## 当前假设

当前 bootloader 提示文本已经明确，因此上位机改为匹配升级提示再发送 `update`，不再对任意串口输出盲发命令。若设备已经输出“开始加载内部FLASH程序”，则判定为错过 7 秒升级窗口并直接报错，方便批量场景快速重试。

## 批量模式说明

- 固件文件在批次开始时选择一次，当前连接会话内复用。
- 断开串口后，本次批量待机结束；重新连接后可重新选择新的固件版本。
- 日志框会实时打印 bootloader 中文提示，便于前期联调和问题定位。
